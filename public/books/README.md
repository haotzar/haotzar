# ספרים לדוגמה - GitHub Pages

תיקייה זו מכילה ספרים שיופיעו באפליקציה כשהיא רצה ב-GitHub Pages.

## 📁 מבנה התיקייה

```
public/books/
├── דוגמה/
│   ├── ספר-דוגמה.txt
│   └── ספר-נוסף.txt
└── README.md
```

## ➕ איך להוסיף ספרים משלך

### קבצי טקסט (.txt)

1. צור תיקייה חדשה (לדוגמה: `תנך`, `משנה`, `תלמוד`)
2. הוסף קבצי טקסט בקידוד **UTF-8**
3. שם הקובץ יהיה שם הספר בממשק

**דוגמה:**
```
public/books/תנך/בראשית.txt
public/books/תנך/שמות.txt
```

### קבצי PDF (.pdf)

1. הוסף קבצי PDF לתיקייה
2. האפליקציה תציג אותם עם PDF viewer מובנה

**דוגמה:**
```
public/books/PDFs/ספר-לדוגמה.pdf
```

## ⚠️ הערות חשובות

### גודל קבצים
- GitHub מגביל קבצים בודדים ל-100MB
- מומלץ לשמור על גודל סביר (עד 10MB לקובץ)
- לספרים גדולים, שקול לפצל אותם לחלקים

### קידוד
- **חובה** לשמור קבצי טקסט בקידוד UTF-8
- אחרת התווים העבריים לא יוצגו נכון

### שמות קבצים
- אפשר להשתמש בעברית בשמות קבצים
- הימנע מתווים מיוחדים: `< > : " / \ | ? *`
- רווחים מותרים, אבל מקף (`-`) מומלץ יותר

## 🔍 איך זה עובד

1. **בזמן Build**: Vite מעתיק את כל התוכן של `public/` ל-`dist/`
2. **בזמן ריצה**: האפליקציה סורקת את `import.meta.glob('/books/**/*.{txt,pdf}')`
3. **בממשק**: הספרים מופיעים ברשימה לפי מבנה התיקיות

## 📝 דוגמת תוכן לקובץ טקסט

```
שם הספר

פרק א - כותרת

תוכן הפרק הראשון...
שורה נוספת...

פרק ב - כותרת נוספת

תוכן הפרק השני...
```

## 🚀 לאחר הוספת ספרים

1. עשה commit ו-push לגיטהאב
2. ה-workflow יבנה מחדש את האתר
3. הספרים יופיעו באפליקציה תוך מספר דקות

## 💡 טיפים

- **ארגון**: צור תיקיות לפי קטגוריות (תנך, משנה, תלמוד וכו')
- **מטא-דאטה**: עדכן את `public/books-metadata.json` עם מידע על הספרים
- **בדיקה מקומית**: הרץ `npm run dev` כדי לבדוק שהספרים נטענים נכון
