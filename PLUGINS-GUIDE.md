# מדריך מערכת התוספים

## סקירה כללית

מערכת התוספים מאפשרת להרחיב את הפונקציונליות של האפליקציה על ידי טעינת תוספים מותאמים אישית. כל תוסף הוא תיקייה עצמאית המכילה קבצי HTML, CSS, JavaScript ואייקון. **התוספים נפתחים בטאבים נפרדים עם אייקון משלהם**, בדיוק כמו כלים אחרים באפליקציה.

## גישה למנהל התוספים

1. לחץ על כפתור **"כלים"** (🔧) בפינה הימנית העליונה של האפליקציה
2. בחר **"תוספים"** מהתפריט הנפתח
3. תגיע לדף מנהל התוספים

## מבנה תוסף

כל תוסף חייב להכיל את המבנה הבא:

```
my-plugin/
├── manifest.json    # קובץ תצורה (חובה)
├── index.html       # קובץ HTML ראשי (חובה)
├── style.css        # קובץ עיצוב (אופציונלי)
├── script.js        # קובץ JavaScript (אופציונלי)
└── icon.png         # אייקון התוסף (מומלץ מאוד - יוצג בטאב!)
```

### קובץ manifest.json

זהו קובץ התצורה של התוסף. דוגמה:

```json
{
  "name": "שם התוסף",
  "version": "1.0.0",
  "description": "תיאור קצר של התוסף",
  "author": "שם המפתח",
  "main": "index.html",
  "icon": "icon.png",
  "styles": ["style.css"],
  "scripts": ["script.js"]
}
```

#### שדות חובה:
- **name**: שם התוסף (מוצג בטאב ובכרטיס התוסף)
- **version**: גרסת התוסף (פורמט: X.Y.Z)
- **main**: נתיב לקובץ HTML ראשי

#### שדות אופציונליים:
- **description**: תיאור התוסף
- **author**: שם המפתח
- **icon**: נתיב לקובץ אייקון (PNG, JPG, SVG) - **מומלץ מאוד!** האייקון יוצג בטאב
- **styles**: מערך של נתיבים לקבצי CSS
- **scripts**: מערך של נתיבים לקבצי JavaScript

### אייקון התוסף

**חשוב:** האייקון יוצג בטאב של התוסף, לכן מומלץ מאוד להוסיף אייקון:
- גודל מומלץ: 32x32 פיקסלים או 64x64 פיקסלים
- פורמטים נתמכים: PNG, JPG, SVG
- האייקון יוצג בגודל 16x16 בטאב
- אם לא מוגדר אייקון, יוצג אייקון ברירת מחדל (🔌)

### קובץ HTML ראשי

קובץ HTML סטנדרטי. דוגמה:

```html
<!DOCTYPE html>
<html lang="he" dir="rtl">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>התוסף שלי</title>
</head>
<body>
    <div class="container">
        <h1>ברוכים הבאים לתוסף שלי</h1>
        <p>זהו תוסף דוגמה</p>
        <button onclick="myFunction()">לחץ כאן</button>
    </div>
</body>
</html>
```

### קבצי CSS

קבצי CSS יוזרקו אוטומטית לתוך ה-`<head>` של ה-HTML:

```css
body {
    font-family: Arial, sans-serif;
    padding: 20px;
    background: #f5f5f5;
}

.container {
    max-width: 800px;
    margin: 0 auto;
    background: white;
    padding: 20px;
    border-radius: 8px;
}
```

### קבצי JavaScript

קבצי JavaScript יוזרקו אוטומטית לפני סגירת ה-`</body>`:

```javascript
function myFunction() {
    alert('התוסף עובד!');
}

// הקוד שלך כאן
console.log('התוסף נטען בהצלחה');
```

## התקנת תוסף

1. פתח את מנהל התוספים (כלים → תוספים)
2. לחץ על כפתור **"הוסף תוסף"**
3. בחר את תיקיית התוסף במערכת הקבצים
4. המערכת תבדוק את תקינות התוסף:
   - קיום קובץ `manifest.json`
   - תקינות השדות החובה
   - קיום קובץ ה-HTML הראשי
5. אם הכל תקין, התוסף יותקן ויופיע ברשימה

## הפעלת תוסף

1. ברשימת התוספים, לחץ על כפתור **"הפעל"** בכרטיס התוסף
2. **התוסף ייפתח בטאב חדש** עם האייקון שלו
3. ניתן לעבור בין טאבים כרגיל
4. ניתן לסגור את הטאב בכל עת

## הסרת תוסף

1. בכרטיס התוסף, לחץ על כפתור **"הסר תוסף"** (🗑️)
2. אשר את ההסרה
3. התוסף יוסר מהרשימה
4. טאבים פתוחים של התוסף יישארו פתוחים עד שתסגור אותם

## אבטחה והגבלות

### Sandbox
כל תוסף רץ בתוך `<iframe>` עם הגדרות sandbox:
```html
<iframe sandbox="allow-scripts allow-same-origin">
```

זה מגביל את התוסף ל:
- ✅ הרצת JavaScript
- ✅ גישה ל-localStorage של התוסף עצמו
- ❌ גישה לאפליקציה הראשית
- ❌ גישה למערכת הקבצים
- ❌ ביצוע בקשות רשת חיצוניות (CORS)

### המלצות אבטחה
- אל תטען תוספים ממקורות לא מהימנים
- בדוק את קוד התוסף לפני ההתקנה
- התוספים נשמרים ב-localStorage של הדפדפן

## דוגמאות לתוספים

### 1. מחשבון פשוט
ראה את התיקייה `example-plugin/` לדוגמה מלאה של תוסף מחשבון עם אייקון.

### 2. רעיונות נוספים
- **כלי המרה**: המרת מטבעות, יחידות מידה, תאריכים
- **משחקים**: סודוקו, טטריס, משחקי חשיבה
- **כלי פרודוקטיביות**: רשימת משימות, טיימר פומודורו
- **מחשבונים מיוחדים**: מחשבון הלכתי, מחשבון גימטריה מתקדם
- **כלי לימוד**: כרטיסיות לימוד, בוחן ידע
- **ויזואליזציות**: גרפים, תרשימים, מפות חום

## טיפים לפיתוח תוספים

### עיצוב אייקון
- השתמש באייקון פשוט וברור
- ודא שהאייקון נראה טוב בגודל קטן (16x16)
- השתמש בצבעים מובחנים
- אייקון SVG יתאים את עצמו אוטומטית

### עיצוב ממשק
- השתמש בעיצוב רספונסיבי
- תמוך במצב כהה אם אפשר
- השתמש בגופנים קריאים
- הוסף padding מספיק לאלמנטים

### ביצועים
- מזער את גודל הקבצים
- השתמש ב-CSS במקום JavaScript כשאפשר
- טען משאבים בצורה עצלה (lazy loading)

## פתרון בעיות

### התוסף לא נטען
- ודא שקובץ `manifest.json` תקין (JSON חוקי)
- בדוק שכל השדות החובה קיימים
- ודא שקובץ ה-HTML הראשי קיים

### האייקון לא מוצג
- ודא שנתיב האייקון נכון ב-manifest
- בדוק שקובץ האייקון קיים בתיקייה
- נסה פורמט אחר (PNG במקום SVG)

### התוסף לא מוצג כראוי
- בדוק את ה-console של הדפדפן לשגיאות
- ודא שנתיבי קבצי CSS ו-JS נכונים ב-manifest
- בדוק שהקבצים קיימים בתיקייה

### JavaScript לא עובד
- ודא שהקוד תקין (בדוק syntax)
- בדוק את ה-console לשגיאות
- זכור שהתוסף רץ ב-sandbox עם הגבלות
